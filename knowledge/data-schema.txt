FitbodGPT Precomputed Analysis Schema

Reports generated by fitbod-report may include precomputed analysis sections. When present, use these values directly instead of computing them via Code Interpreter.

---

## analysis

TSV section with metric/value pairs. One row per metric.

Fields:

push_pull_ratio: Float. Push working sets / pull working sets. Target 1.0-1.5 favoring pull.
upper_lower_ratio: Float. Upper body weekly avg sets / lower body weekly avg sets.
lower_share_pct: Float. Lower body sets as percentage of total upper+lower. Target 30-40%.
level_score: Float 0-100. Weighted score from 7 axes (see coaching-guidelines.txt for formula).
level: String. "Beginner" (0-33), "Intermediate" (34-66), "Advanced" (67-100).
stalled_compounds: Comma-separated "Name:trend%" or "none". Compound exercises with trend <= 0% and 4+ sessions.
stalled_isolations: Comma-separated "Name:trend%" or "none". Isolation exercises, same criteria.
first_half_sessions_pw: Float. Average sessions per week in first half of date range.
second_half_sessions_pw: Float. Average sessions per week in second half.
first_half_vol_pw: Float. Average weekly volume (weight*reps) in first half.
second_half_vol_pw: Float. Average weekly volume in second half.
volume_drop_pct: Float or "N/A". Percentage change from first to second half. Negative means drop.
gap_weeks: Comma-separated ISO dates (week start) or "none". Weeks with no training data in the report.
unknown_exercise_count: Integer. Custom/unrecognized exercises excluded from ratio calculations. If > 0, ratios are approximate.
cardio_undercount: "true" or "false". True when all period cardio times are zero but the last 5 periods contain at least one cardio exercise (e.g., Running, Walking).

---

## equipment

One equipment name per line (no header row). Human-friendly names sorted by usage volume. Only includes equipment used in working sets.

Example:
machines
dumbbells
barbell
cable station
pull-up bar

---

When these sections are present:
  Use the precomputed values for the initial analysis summary. You still need to read the period_summary, exercise_stats, and muscle_volume sections for the detailed numbers. If anything looks unusual or inconsistent (e.g., a ratio that doesn't match what you see in the raw data), use Code Interpreter to cross-check before presenting it. The precomputed values are a starting point, not a substitute for judgment.

When these sections are absent:
  The user uploaded a raw CSV export or TXT file directly from Fitbod, or the report was generated without the precomputation option enabled. Compute everything via Code Interpreter using these steps:

  1. Load exercise-database.json. For each exercise in the report, look up its movement_pattern, primary_muscles, equipment, is_compound, and category.
  2. Filter to strength exercises only (category = "strength" or "plyometric"). Skip cardio and mobility.
  3. Push:pull ratio: sum working_sets where movement_pattern is horizontal_push/vertical_push/isolation_push (push) vs horizontal_pull/vertical_pull/isolation_pull (pull). Divide push by pull.
  4. Upper:lower ratio: use the muscle_volume section's weekly_avg_sets (NOT working_sets from exercise_stats). Sum for upper muscles (chest, upper_chest, lats, upper_back, shoulders, front_delts, side_delts, rear_delts, biceps, triceps, traps, forearms) vs lower muscles (quads, hamstrings, glutes, calves, adductors, hip_flexors).
  5. Level score (7-axis weighted, see coaching-guidelines.txt for full formula):
     - Exercise variety (15%): <6=0, 6-15=5, >15=10 unique strength exercises
     - Training frequency (15%): <2/wk=0, 2-4=5, >4=10
     - Compound lift count (15%): 0-1=0, 2-4=5, 5+=10
     - Overload evidence (20%): % of compounds with positive trend, mapped to 0-10
     - Volume per session (15%): total working sets / total sessions; <8=0, 8-16=5, 16+=10
     - Data depth (10%): <4wk=0, 4-26=5, 26+=10
     - Sophistication (10%): % of sets on free weights (barbell/dumbbell/kettlebell/ez_bar) or unilateral, mapped to 0-10
     - Final: sum(axis * weight) * 10. 0-33=Beginner, 34-66=Intermediate, 67-100=Advanced.
  6. Stalled exercises: filter for sessions >= 4, trend <= 0%, strength category. Split into compounds and isolations. Sort each group by trend ascending (most negative first).
  7. Training gaps: only detect for weekly reports (period intervals of 5-9 days). Compare period start dates to expected week starts in the date range. Missing dates are gaps. Skip gap detection for monthly/quarterly/yearly reports.
  8. Volume drop: split the reported periods (NOT the full date range) into first half and second half. Do NOT zero-fill missing weeks — only average the periods that exist in the report. Use total_volume from each period's stats. Compare avg sessions and avg volume between halves. Flag if drop > 30%. Note: for non-weekly period types, the values represent per-period averages, not per-week.
  9. Equipment: collect unique equipment types from exercises with working_sets > 0. Map to human-friendly names.
  10. For exercises not in the database, infer muscle groups from the name and track them as unknown.
  11. Cardio detection (for raw CSV): an exercise is cardio if Duration > 0 AND Distance > 0, OR if Reps == 0 AND Duration > 300. Sum Duration(s) for cardio exercises to get cardio minutes. Sum Duration(s) for non-cardio exercises to get strength minutes. If the report has a period_summary with cardio_min column, use those values directly.
  12. Cardio undercount check: if all cardio_min values are 0 but recent_sessions show cardio exercises (Running, Walking, Cycling), flag it. This can happen with older report versions.
